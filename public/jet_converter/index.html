Documentazione
Guida all'utilizzo del portale — dal file grezzo all'output pronto per l'importazione in Jet.

Come funziona

Il portale trasforma i file dati grezzi dei clienti — tipicamente esportati dal gestionale precedente — nel formato standard richiesto da Jet: il template anagrafica base con 59 colonne.

Il flusso si articola in quattro fasi sequenziali. Completare una fase prima di passare alla successiva garantisce il risultato corretto.

1
Upload
Carica il file .xlsx grezzo del cliente. Il portale verifica che abbia le colonne obbligatorie e lo salva nello storico.
2
Validazione
Il sistema controlla ogni campo (codici fiscali, P.IVA, email, CAP, duplicati) e propone correzioni automatiche. Puoi accettare o rifiutare ogni suggerimento.
3
Arricchimento
Opzionale. Completa automaticamente i codici ISTAT dei comuni mancanti usando i dati integrati dal template Jet. Se qualche comune non viene trovato, è disponibile un fallback online.
4
Output
Visualizza la tabella finale, compila manualmente i campi mancanti (CCNL, orario, sede ecc.), poi esporta il file .xlsx pronto per l'importazione in Jet.
I dati rimangono nel browser (localStorage). Nulla viene inviato a server esterni.

1. Upload

Trascina il file .xlsx nell'area tratteggiata, oppure cliccaci sopra per aprire il selettore file. Il portale accetta il file solo se contiene tutte le colonne obbligatorie:

Codice_fiscale  ·  Nome  ·  Cognome  ·  Codice fiscale azienda  ·  Denominazione azienda  ·  Indirizzo_email

Se mancano colonne obbligatorie, appare un messaggio con l'elenco preciso di quelle assenti. Il file non viene salvato finché non è conforme.

I file caricati appaiono nello Storico Upload. Puoi filtrarli per nome file o per data di caricamento, visualizzare un'anteprima dei dati con il tasto Preview, o eliminarli singolarmente. Puoi caricare file da fonti diverse e lavorarli insieme nella validazione.

Se carichi di nuovo un file già presente (stesso nome e stesso numero di righe) viene ignorato per evitare duplicati nello storico.

2. Validazione

Seleziona uno o più file dallo storico (metti il segno di spunta accanto al nome) e premi Avvia validazione. Il sistema analizza tutti i record dei file selezionati.

Errori critici e avvisi

I problemi trovati si dividono in due categorie:

Errori critici

La riga viene esclusa dall'output.

  • Codice Fiscale non valido (MOD-26)
  • P.IVA non valida (Luhn)
  • Campi obbligatori mancanti
  • Valori non ammessi nei campi a selezione
Avvisi

La riga viene inclusa, ma segnalata.

  • Email con formato non standard
  • CAP non a 5 cifre
  • Provincia di residenza assente
  • Spazi all'interno di un codice

Correzioni suggerite

Quando il sistema trova un problema, apre una finestra con l'elenco. Per ogni voce puoi scegliere Mantieni originale oppure Accetta correzione (disponibile solo quando il sistema riesce a calcolarla automaticamente, ad esempio rimuovendo uno spazio da un CF). I pulsanti in cima alla finestra permettono di accettare o rifiutare tutte le correzioni in blocco.

Duplicati

Dopo aver gestito le correzioni, il sistema cerca record duplicati confrontando Codice Fiscale, P.IVA e indirizzo email. Per ogni coppia di duplicati puoi scegliere:

  • Mantieni separati — entrambi i record rimangono nell'output
  • Mantieni esistente — il record nuovo viene ignorato
  • Usa nuovo — il record precedente viene sostituito
  • Annulla — nessuna azione automatica; gestisci la situazione a mano in Output

3. Arricchimento (opzionale)

Il portale include già i dati di tutti i comuni italiani e delle nazioni, caricati direttamente dal template Jet. Per ogni dipendente cerca automaticamente il nome del comune di residenza e ricava il codice ISTAT necessario per il campo sede_di_lavoro__comune.

Se un comune non viene trovato nel dataset integrato, puoi avviare il fallback online: il portale scarica il dataset pubblico ISTAT (~8.000 comuni) e ritenta la ricerca. È necessaria la connessione internet solo per questo passaggio.

Se la tua versione del template Jet ha comuni o nazioni aggiornate rispetto a quelle integrate, usa il pulsante Carica nuovo template per aggiornare il dataset senza ricaricare la pagina.

Se non hai bisogno dell'arricchimento (ad esempio perché compilerai i campi sede direttamente in Output) puoi saltare questa sezione e andare direttamente a Output.

4. Output

La tabella mostra tutti i record mappati nelle 59 colonne del template Jet. Le righe evidenziate in rosso contengono errori che impediscono l'importazione. Le righe su sfondo bianco sono pronte.

Filtrare le colonne

Usa le pillole in cima (cliente, persona, rapporto, variazione, sede, orario) per mostrare o nascondere gruppi di colonne. Sotto ogni intestazione c'è un campo di ricerca testuale — i filtri su colonne diverse si sommano tra loro.

Modificare i dati

Clicca su qualsiasi cella per modificarla direttamente. Le modifiche vengono salvate automaticamente nel browser. Compila qui i campi non presenti nel file di partenza: tipologia contratto, CCNL, orari, sede di lavoro, ecc.

Per cancellare esplicitamente un valore già presente in Jet, scrivi $VUOTO$ nella cella. Una cella vuota viene ignorata in importazione (non sovrascrive nulla); $VUOTO$ invece svuota attivamente il campo.

Esportare

  • Esporta validi (.xlsx) — solo le righe senza errori, nel formato pronto per Jet (foglio "Import definitivo")
  • Esporta selezionati (.xlsx / .csv) — solo le righe con il segno di spunta attivo
  • Scarica con errori (.xlsx) — le righe problematiche con una colonna aggiuntiva che descrive ogni errore, per revisione offline
